# WhatsApp Transcription Bot

WhatsApp bot that automatically transcribes received voice messages. The system runs inside a Docker container and uses Google servers for speech transcription.

The project idea was born based on this repository: <br>
https://github.com/puluceno/WhatsappTranscriptionOffline <br>
Almost all the code was generated by DeepSeek; I mostly acted as a tester for debugging and orchestrated its work.

---

## Overview

This project implements a WhatsApp bot that listens for incoming voice messages, converts them to the appropriate audio format, and transcribes them using speech recognition services. The transcription is then sent back as a reply to the original message.

---

## System Architecture

### Operating System

- **Ubuntu 22.04 LTS (Jammy Jellyfish)** - Stable, long-term supported Linux distribution

### Main Components

The container includes three main components that cooperate with each other:

**1. WhatsApp Client (Node.js)**

- File: `index.js`
- Library: `whatsapp-web.js` with Puppeteer
- Purpose: Interface with WhatsApp Web, receive messages, handle QR code

**2. Transcription Server (Python)**

- File: `server/app.py`
- Framework: FastAPI with Uvicorn
- Purpose: Receives audio files, sends them to Google servers for transcription, adds punctuation

**3. Process Manager (Supervisord)**

- File: `supervisord.conf`
- Purpose: Starts and monitors Node.js and Python processes, restarts them in case of crash

> **Note:**  All folders under `Volumes/` are mounted as Docker volumes to specific container paths. This keeps critical data (session, models, cache) directly on the Windows filesystem, avoiding uncontrolled growth of the WSL2 VHDX file.  
The entire project directory (excluding `node_modules`) is bind-mounted to `/app` inside the container. This allows you to edit source files (`.env`, `index.js`, `server/app.py`, etc.) and apply changes by simply restarting the container with `docker-compose restart`. No rebuild is needed unless you modify `package.json` or `requirements.txt`.

## Configuration

### Environment Variables

To prevent the bot from transcribing audio in a specific WhatsApp group:

1. **Temporarily enable ID display**  
   Edit the `.env` file and set `SHOW_CHAT_IDS=true`.  
   Then restart the container:  
   ```bash
   docker-compose restart
   ```

2. **Find the group ID**  
   Send any message (text is enough) in the group you want to exclude.  
   In the container logs (`docker-compose logs -f`), you will see a line like:  
   ```
   [DEBUG] Messaggio da chat: 1234567890-123456@g.us (isGroup=true) - Tipo: chat
   ```  
   The part `1234567890-123456@g.us` (or ending with `@lid`) is the group ID.

3. **Set the exclusion**  
   Open `.env` again and add the ID to `EXCLUDED_GROUPS`. For example:  
   ```
   EXCLUDED_GROUPS=1234567890-123456@g.us
   ```
   To exclude multiple groups, separate them with commas:  
   ```
   EXCLUDED_GROUPS=id1@g.us,id2@g.us
   ```

4. **Restart the container**  
   After editing `.env`, restart the container:  
   ```bash
   docker-compose restart
   ```

5. **(Optional) Hide IDs again**  
   If you prefer not to show real IDs in logs, set `SHOW_CHAT_IDS=false` in `.env` and restart the container once more.

## How It Works

### Processing Flow

1. **Reception**: The bot receives a voice message on WhatsApp
2. **Download**: Downloads the audio file in OGG/Opus format
3. **Conversion**: Converts to 16 kHz mono WAV with ffmpeg
4. **Send to Google**: The Python server sends the audio file to Google Speech Recognition servers
5. **Receive transcription**: Receives the transcribed text from Google servers
6. **Punctuation**: Adds punctuation with a local NLP model
7. **Reply**: Sends the transcription as a reply to the message
8. **Cleanup**: Deletes temporary files

### Privacy and Security

**Warning**: Audio files are sent to Google servers for transcription.

- No permanent storage of audio files in the container
- No logging of transcriptions
- Logs contain only technical information (duration, timings)
- Temporary audio files are deleted after processing
- The WhatsApp session is saved locally in the `Volumes/session_data/` folder

## Optimizations and Maintenance

### Disk Growth Control

To prevent the WSL2 VHDX file from growing indefinitely, the following measures have been adopted:

- Windows volumes: All directories that accumulate data (session_data, model cache, logs, etc.) are mounted to Windows folders via `./Volumes/`.
- Files in `/tmp` older than 60 minutes are deleted every hour.
- Every 24 hours, Hugging Face temporary files not accessed for 30 days (e.g., `.lock` files) are removed while preserving main models.
- Docker log rotation: `docker-compose.yml` is configured with a limit of 3 log files of 100 MB each.
- VHDX size limit: Create (or modify) the file `%UserProfile%\.wslconfig` with the following content at the top:

```
[wsl2]
defaultVhdSize=20GB
```

### Shrink VHDX File

After starting and configuring the container, you can reclaim disk space by compacting the Docker distro VHDX using these commands

```bash
docker system prune -a -f --volumes

docker builder prune -a -f
```

Close Docker from the TrayBar

```bash
wsl --shutdown

diskpart

select vdisk file="%LocalAppData%\Docker\wsl\disk\docker_data.vhdx"

attach vdisk readonly

compact vdisk

detach vdisk

exit
```
